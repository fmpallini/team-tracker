# Team Tracker v1.1 — Feedback round 1 (design)

Data: 2026-07-05. Origem: feedback do usuário após uso do v1.0.0. Aprovado integralmente nesta data.

Escopo: 9 frentes — polish visual, onboarding, organograma, editor, módulos (riscos/actions/milestones), calendário, ajuda global, busca — incluindo 3 bugs. Uma migração de schema (v1→v2).

## Decisões tomadas (com o usuário)

- **Emoji picker:** picker próprio (grid curado por categorias) + hint do atalho do SO (`Win+.` / `Cmd+Ctrl+Space`). Campo continua texto livre.
- **Split inicial de time:** pane direito abre o módulo **Members** (árvore).
- **Organograma:** CSS puro top-down (caixas HTML, linhas via pseudo-elementos). DnD nativo preservado.
- **Highlight de busca:** CSS Custom Highlight API; fallback = scroll até o primeiro match.
- **GitHub:** `https://github.com/fmpallini/team-tracker`.

## 1. Tela inicial

Abaixo do título, tagline + 3 bullets de vantagens (i18n pt-BR/en-US):

- **100% offline** — funciona sem internet, nada sai da sua máquina.
- **Seus dados, seu controle** — um único arquivo `.tmv` que você guarda onde quiser.
- **Criptografia de ponta a ponta** — AES-256, senha só sua.

Botões com emojis: ⏪ Reabrir último, 📂 Abrir, ✨ Criar.

## 2. Visual geral (styles.css)

- Bordas/contornos em cards, módulos e áreas de interesse.
- Sombras suaves: elevação em modais, dropdowns, caixas do organograma.
- Hover states mais evidentes; destaque do pane focado.
- Emojis unicode em botões principais: 📋 templates, ❓ ajuda, ⚙️ preferências, 🔍 busca, ➕ adicionar. Toolbar do editor mantém glifos B/I/U/S.
- Tema claro/escuro preservado em tudo.

## 3. Onboarding

- **Arquivo sem times:** pane central mostra convite "👋 Crie seu primeiro time" + botão que abre o modal de novo time (substitui o pane vazio genérico).
- **Emoji picker:** popup ao focar o campo emoji (add/edit time), grid com categorias (pessoas, símbolos, objetos, bandeiras…), hint do atalho do SO no rodapé.
- **Primeira abertura de um time:** split automático — esquerda = notas do dia (hoje), direita = árvore de members. "Primeira" = pane history vazio para aquele time.
- **Sidebar:** hint visível do atalho `Alt+1`..`Alt+9` por item de time (badge discreto ou tooltip).

## 4. Organograma (stakeholders/members)

- Layout top-down: caixa por pessoa — nome em uma linha, cargo em fonte menor + itálico na segunda.
- Linhas conectoras via pseudo-elementos CSS; irmãos lado a lado no mesmo patamar; scroll horizontal quando necessário.
- Drag-and-drop atual (mudar subordinação/ordem) preservado.
- **Duplo clique** na caixa abre as notas da pessoa; clique simples continua selecionando/arrastando.

## 5. Editor

- Botões novos na toolbar: **¶** parágrafo normal (`formatBlock <p>`) e **🧹** limpar formatação (`removeFormat`).
- Autocomplete `@`: itens relativos **@hoje / @ontem / @amanhã** (en: @today / @yesterday / @tomorrow) — inserem chip de dia com a data resolvida no momento da inserção.
- **BUG `/`:** picker de templates abre mas clicar/Enter não insere. Root-cause com systematic-debugging (suspeita: range do trigger stale em `commit()`); teste de regressão antes do fix.
- Help do editor: remover seção `chrome --app` (migra para o help global, §8).

## 6. Módulos — schema v2

Migração v1→v2 em `core/document.ts` (`MIGRATIONS[1]`), com teste sobre doc v1 real:

- **Risk** ganha `closed: boolean` (default `false`).
- **ActionItem** ganha `notes: string` (default `''`).
- **Milestone** ganha `followup: string` (default `''`). (Usuário citou followup de milestones como existente — hoje só riscos têm; passa a existir.)

Comportamento:

- **Riscos:** botão ✔️ concluir por linha → risco vai para seção colapsável "Riscos concluídos" no fim do módulo, com botão de reabrir. Riscos fechados fora da tabela principal de exposição.
- **Action items:** nota livre expansível por item — mesmo padrão do followup de riscos (editor rico + `@` + `/`, scope 'any').
- **Milestones:** followup expansível por item, mesmo padrão.
- Resultado: **todo campo livre** do app (daily, person notes, followup de riscos, followup de milestones, notas de action items) tem editor rico com `@` e `/`.

## 7. Calendário

- Três estados visuais independentes e combináveis: **hoje** = contorno/anel; **com nota** = fundo tingido; **selecionado** = fundo sólido.
- **Dois dias em split:** abrir daily em cada pane com dias diferentes tem que funcionar (requisito original; hoje falha). Diagnosticar (suspeita: nav/calendário sobrescreve o outro pane ou `Loc` compartilha data), corrigir, teste cobrindo o cenário.

## 8. Ajuda global + Sobre

- Botão ❓ no shell (barra superior) → modal de ajuda global: `Alt+1..9` (times), `Ctrl+K` (paleta), `Ctrl+S` (salvar), `Alt+←/→` (histórico), `F11` (fullscreen do browser); seção "janela de app" com a receita `chrome --app` movida do help do editor; ponte para o help do editor.
- Preferências → aba Sobre: link para `https://github.com/fmpallini/team-tracker`.

## 9. Busca

- **BUG clique:** clicar num resultado não navega para o módulo. Diagnosticar (handler ou re-render engolindo o clique), teste de regressão, fix.
- **Highlight:** ao abrir resultado, pintar os termos via CSS Custom Highlight API (`CSS.highlights` — não altera DOM, não suja o contenteditable nem o dirty state). Limpa ao editar/navegar. Fallback sem suporte: scroll até o primeiro match.
- **Cobertura do índice** (`core/search.ts`): incluir campos novos (`ActionItem.notes`, `Milestone.followup`); revisão dos existentes — daily ✓, person notes ✓, action text+assignee ✓, milestone title ✓, risk title+followup ✓.
- **Âncora por item:** resultados de riscos/actions/milestones apontam o item específico (não só o módulo), para o highlight rolar até ele. Extensão do `ModuleRef` desses módulos com id opcional do item. `ModuleRef` é persistido no nav do `.tmv`, mas campo opcional novo não exige migração (docs antigos simplesmente não o têm); entra junto no bump v2 de qualquer forma.
